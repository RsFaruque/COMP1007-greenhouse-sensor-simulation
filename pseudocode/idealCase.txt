In an ideal case, reading and building CsvData would be separated. This is to make CsvData more generic.

CSV class would read the CSV file and convert each record into a data type. The data type can be specified by passing the class as an argument. This allows the CsvData to hold any type of data.

To avoid ambiguity and seamless integration, developers must implement CsvDataType interface in their classes.

    public static CsvData<T> readCSV (String filePath);
